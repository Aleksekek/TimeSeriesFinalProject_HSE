# Прогнозирование временных рядов: Corporación Favorita Grocery Sales Forecasting

## Описание проекта

Проект посвящен прогнозированию продаж для сети супермаркетов Corporación Favorita в Эквадоре. Задача состоит в предсказании ежедневных продаж тысяч товаров в разных магазинах на основе исторических данных с учетом различных внешних факторов.

**Основные характеристики данных:**
- 125+ миллионов записей о продажах с 2013 по 2017 год
- 54 магазина, объединенных в 17 кластеров
- 4100+ товаров, разделенных на семейства
- Наличие внешних признаков: цены на нефть, праздники (не использовались), данные по транзакциям

**Метрика качества:** NWRMSLE (Normalized Weighted Root Mean Squared Logarithmic Error) с весами 1.25 для скоропортящихся товаров.

## Эксперименты и результаты

### 1. Предобработка данных

**Проблемы и решения:**
- **Пропуски в onpromotion**: обнаружено 100% пропусков в первые 15 месяцев → удаление этого периода
- **Неявные пропуски дней**: отсутствие записей для дней без продаж → заполнение нулями
- **Оптимизация памяти**: функция reduce_mem_usage сократила использование памяти с 8GB до 1.78GB

### 2. Разведочный анализ (EDA)

**Ключевые выводы:**
- Ярко выраженная недельная сезонность (подтверждена ACF/PACF)
- Пик продаж в декабре (предновогодний сезон)
- Доля промо резко выросла с ~3% до ~12% после января 2016
- Отношение сигнал/шум ~2 (шумная компонента ±5 при среднем 10)

### 3. Базовые модели

| Подход | NWRMSLE | Комментарий |
|--------|---------|-------------|
| Константа 0 | 1.916 | Худший результат |
| Глобальное среднее | 1.028 | Наивный baseline |
| **Naive Seasonal (7 дней)** | **0.705** | Учитывает недельную сезонность |
| AutoETS (иерархический) | 1.421 | Проблемы с масштабированием |

Naive Seasonal показал хороший результат благодаря выраженной сезонности, но иерархический подход с AutoETS столкнулся с проблемами масштабирования при дизагрегации.

### 4. Прямые модели на топ-50 парах

Для более чистого сравнения моделей были отобраны топ-50 самых популярных пар магазин-товар.

| Модель | NWRMSLE | Время обучения | Комментарий |
|--------|---------|----------------|-------------|
| **AutoETS Direct** | 0.543 | ~2 мин | Хороший статистический baseline |
| **CatBoost Direct** | 0.493 | ~10 мин (GPU) | ML-подход с признаками |
| **DLinear** | **0.482** | ~1 мин (CPU) | Победитель по качеству/скорости |

**Наблюдения:**
- DLinear показал лучшее качество при минимальном времени обучения
- CatBoost дал близкий результат, но требовал больше ресурсов
- AutoETS - достойный статистический baseline

### 5. Иерархический подход (бонус)

В качестве эксперимента был реализован иерархический пайплайн с агрегацией по трем уровням:
- Total (общий уровень)
- Cluster (17 кластеров магазинов)
- Cluster+Family (~550 групп)

**Результаты:**
- AutoETS (иерархический): 1.421
- CatBoost (иерархический **с внешними признаками**): 1.274

**Вывод:** иерархический подход упростил вычисления (570 моделей вместо 170k), но проиграл в качестве из-за потери индивидуальных особенностей рядов при агрегации (решается усложнением логики кластеризации).

## Ключевые выводы

1. **Простота побеждает**: DLinear, несмотря на простую архитектуру, показал лучшее качество и скорость
2. **Масштабирование - главная проблема**: иерархический подход решил проблему вычислений, но проиграл в точности
3. **Качество данных важнее сложности моделей**: заполнение пропусков и чистка данных дали больше, чем усложнение архитектур
4. **Дополнительные признаки помогают, но не критично**: CatBoost с признаками показал улучшение, но DLinear обошелся без них
